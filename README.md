# Marist Water Polo — Registration App

A club registration web app for Marist Water Polo (Auckland). Manages player sign-ups, displays registered members, and assigns players to teams.

Built with React 19 + Vite. Registration data is persisted to a Supabase (PostgreSQL) database.

---

## What the app does

- **Two registration modes** — a choice screen routes registrants to either Family Registration (parent-led, multi-player) or Individual Registration (adult 18+ registering themselves)
- **Family registration** — parent or guardian details are collected first, then players are added one at a time with a live family panel showing registrations as they are added; supports a second parent/guardian
- **Individual registration** — simplified single-form flow for adult players; shows a warning and switch link if the entered date of birth is under 18
- **Auto-calculates grade** from date of birth and gender, based on age as at 1 January of the current year (Under 12s through to Senior)
- **Unique club registration ID** — every registration is assigned a sequential ID in the format `MWP-YYYY-NNNN` (e.g. `MWP-2026-0001`), generated by the database and displayed in the Members table
- **Address autocomplete** — if an AddressFinder API key is configured, the street address field offers live NZ address suggestions and auto-fills suburb, city, and postcode on selection; falls back to a static suburb lookup if no key is set
- **Parent as player** — each parent section includes a Player Status toggle; parents who are also players complete their own registration details and are saved as separate member records with a "Parent-Player" badge
- **Volunteer sign-up** — parents can indicate they want to volunteer (Committee, Manager, Coach, or Other) during registration
- **Enforces age rules** — players under 21 must provide a school or tertiary institution; players under 18 must use Family Registration so parent details are collected
- **Lists all registered members** in a table with Reg ID, photo, grade, contact details, and parent information
- **Assigns players to squads** on the Teams page using a dropdown per player
- **Admin page** for editing the terms and conditions shown on the registration form

## Environment setup

Copy `.env.example` to `.env` and fill in your values:

```
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-publishable-key-here
VITE_ADDRESSFINDER_KEY=your-addressfinder-key-here   # optional
```

- **Supabase** — required for data persistence. Without it the app runs in local-only mode (yellow warning banner, data lost on refresh).
- **AddressFinder** — optional. Sign up at [addressfinder.nz](https://addressfinder.nz). Without it the address section falls back to a static NZ suburb lookup.

Never commit `.env` — it is listed in `.gitignore`.

---

## Database setup (Supabase)

1. Create a free project at [supabase.com](https://supabase.com)
2. In the Supabase dashboard go to **SQL Editor** and run the full contents of `supabase-setup.sql`
3. This creates the `members` table, the `reg_id_counters` table, the `generate_reg_id()` function, and the trigger that auto-assigns `MWP-YYYY-NNNN` IDs on every insert
4. Add your project URL and publishable (anon) key to `.env`

## Running locally

**Prerequisites:** Node.js 18+

```bash
# Install dependencies
npm install

# Start the development server
npm run dev
```

The app runs at `http://localhost:5173` by default.

```bash
# Other commands
npm run build     # Production build → dist/
npm run preview   # Serve the production build locally
npm run lint      # Run ESLint
```

## Grade system

Grades are determined by the player's age on 1 January of the current calendar year:

| Age on 1 Jan | Male | Female | Other |
|---|---|---|---|
| Under 12 | Under 12s | Under 12s | Under 12s |
| 12–13 | Under 14 Boys | Under 14 Girls | Under 14s |
| 14–15 | Under 16 Boys | Under 16 Girls | Under 16s |
| 16–17 | Under 18 Men | Under 18 Women | Under 18s |
| 18–20 | Under 21 Men | Under 21 Women | Under 21s |
| 21+ | Senior Men | Senior Women | Senior |

## Contributing

1. Fork the repository and create a branch from `main`
2. Copy `.env.example` to `.env` and configure your Supabase credentials
3. Run `npm run lint` before submitting — the CI check will fail on lint errors
4. All styles are inline JS objects defined at the top of each component file — follow this pattern rather than adding CSS files
5. Shared app state (members list, terms text) lives in `App.jsx` and is passed as props — do not introduce a state management library without discussion
6. Submit a pull request with a clear description of what changed and why

## Known limitations

- **Team assignments** reset when navigating away from the Teams page, as that state is local to the component.
- **No authentication** — the Admin page is publicly accessible.
- **Photo storage** — player photos are stored as base64 strings in the database. For clubs with many photo uploads, migrating to Supabase Storage would be more efficient.
- **NZ suburb fallback coverage** is ~240 suburbs, weighted toward Auckland. Configure AddressFinder for full NZ address coverage.
